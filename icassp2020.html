A Hierarchical Tracker for Multi-Domain Dialogue State Tracking

J. Li, S. Zhu and K. Yu, ICASSP 2020 - 2020 IEEE International Conference on Acoustics, Speech and Signal Processing (ICASSP), Barcelona, Spain, 2020, pp. 8014-8018, doi: 10.1109/ICASSP40776.2020.9053248.[Paper]

Abstract

The goal of Dialogue State Tracking (DST) is to estimate the current dialogue state given all the preceding conversation. Due to the increased number of state candidates, data sparsity problem is still a major hurdle for multi-domain DST. Existing methods generally choose to predict a value for each possible slot over all domains with quite low efficiency. In this paper, we propose a hierarchical dialogue state tracker which consists of three sequential modules: domain classification, slot detection and value extraction. It predicts domains, slots and values dynamically by given the dialogue history and outputs of the preceding module, which can dramatically improve the model efficiency. Experimental results on MultiWOZ2.1 also show that our approach achieves state-of-the-art joint goal accuracy, and confirm that the hierarchical structure can enhance existing DST models significantly.

Introduction

In a task-oriented spoken dialogue system (SDS), the control module between input and output is referred as dialogue management. It comprises two components, inferring the user intentions and giving the responses. For the former, Dialogue state is used to recording the user goals for the given slots at each dialogue turn and often represented as slot-value pair. Dialogue state tracking (DST) aims to track and update the dialogue state across dialogue turns. Given the dialogue state of current turn, a suitable dialogue policy (DP) approach would provide a proper response.

Different from single domain DST task (e.g., DSTCs), dialogue states in multi-domain DST is often formed as a set of domain-slot-value triplets. With the development of the multi-domain dialogue dataset (MultiWOZ) recently, sharing parameters across different domains or slots become a convention due to the increase number of slots. These methods try to predict the value for each possible domain-slot pair one by one, which shows quite low efficiency.

Actually, each dialogue state can be reconstructed using a hierarchical structure as shown in Fig.1. The value for a domain-slot pair not mentioned is settled as None. Based on these properties, we split the multi-domain DST task into three sequential parts: domain classification, slot detection and value extraction. The task decomposition leads to faster training and inference speeds, and helps alleviate the data sparsity problem since dialogue states are also separated into three parts.

For domain and slot models, we adopt recurrent neural networks (RNNs) to encode utterances and dialogue history into a deep representation for classification. In the value model, pointer network is applied to predict value start and end indexes for a slot. Additionally, it is such a portable approach that different value models can be integrated with the domain and slot models. The existing DST models, which try to predict values for all domain-slot pairs, are also investigated to server as the value model. Experimental results on MultiWOZ2.1 show that our approach can achieve state-of-the-art joint goal accuracy, and also enhance the existing DST models significantly.
